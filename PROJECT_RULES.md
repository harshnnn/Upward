# Upward Project Rules

## General

- TypeScript everywhere
- No any types
- Strict typing enabled
- Production-quality code only
- No placeholder implementations
- Reusable architecture
- Feature-first structure where appropriate
- Shared types between backend/web/mobile

## Frontend

- React for web
- React Native + Expo for mobile
- Tailwind for web
- NativeWind for mobile
- Responsive design
- Dark mode first
- Accessible UI components
- Reusable hooks
- Reusable UI primitives

## Backend

- NestJS
- Modular architecture
- Prisma ORM
- PostgreSQL
- JWT authentication
- Proper validation
- Global error handling
- Structured logging

## Database

- UUID primary keys
- createdAt and updatedAt everywhere
- soft delete where appropriate
- optimized indexes
- scalable schema design

## APIs

- RESTful APIs
- Versioned endpoints
- Proper DTO validation
- Consistent response format
- Pagination support

## Code Quality

- ESLint
- Prettier
- Husky
- Conventional commits

## Performance

- Avoid unnecessary re-renders
- Memoize expensive operations
- Optimized queries
- Lazy loading where useful

## Scalability

- Millions of records support
- Analytics-ready architecture
- Event-driven opportunities considered
- Background jobs architecture ready
